### 3. Модель аренды: Копи-трейд с Leader-счёта

ПО предоставляется Пользователю **в аренду**. Арендная плата взимается по модели «Копи-трейд»:

#### 3.1 Структура счетов
- Пользователь имеет **1 (один) основной leader-счёт** и до **4 (четырёх)** дополнительных копи-счетов
- Все счета подключены к ПО в режиме копи-трейда (copy-trade)
- **Один из копи-счетов всегда является счётом Автора** (далее — «Счёт Автора»)

#### 3.2 Принцип копи-трейда
- Leader-счёт определяет торговые решения; все копи-счета **зеркально повторяют сделки leader-счёта**
- Все входы, выходы, стоп-лоссы и тейк-профиты идентичны на всех счетах
- Если leader-счёт заработал $3,000 — каждый копи-счёт тоже заработал $3,000
- Счёт Автора не может быть отключён от копи-трейда, пока активно подключение ПО

#### 3.3 Арендная плата
- **Прибыль, заработанная на Счёте Автора, является арендной платой за использование ПО**
- Пользователь НЕ вносит фиксированную ежемесячную плату
- Пользователь НЕ компенсирует убытки Счёта Автора из собственных средств
- Если на Счёте Автора образовался убыток — арендная плата за этот период считается равной нулю
- **Пример:** Leader-счёт заработал $3,000. Каждый копи-счёт (включая Счёт Автора) также заработал $3,000. Автор получает $3,000 арендной платы. Пользователь сохраняет прибыль leader-счёта ($3,000) + прибыль своих копи-счетов (до 3 × $3,000 = $9,000). Итого Пользователь: до $12,000, Автор: $3,000.

#### 3.4 Обязательства Пользователя
- Пользователь обязуется обеспечить ПО доступом ко всем счетам (включая Счёт Автора) на протяжении всего срока использования ПО
- Пользователь обязуется не отключать Счёт Автора от копи-трейда
- Пользователь обязуется не создавать отдельных экземпляров ПО для обхода Счёта Автора
- Пользователь обязуется не модифицировать ПО с целью исключения Счёта Автора из копи-трейда

#### 3.5 Ответственность за нарушение
Нарушение условий раздела 3.4 влечёт:
- Немедленное прекращение доступа к ПО
- Обязательство компенсировать Автору упущенную прибыль за весь период использования ПО
- Возможное судебное преследование в соответствии с применимым правом

### 4. Описание сервиса

**Jafar Pro** — это программный комплекс для анализа финансовых рынков, включающий:

- **AI-Ассистент** — аналитический инструмент на базе искусственного интеллекта (DeepSeek, Claude Opus, Gemini, Ollama), предоставляющий торговые рекомендации, анализ рыночной структуры (SMC/ICT), уровни поддержки/сопротивления, и оценку волатильности
- **Торговые боты (Blade, Autobot)** — автоматизированные системы анализа и исполнения торговых операций через API брокера TopstepX
- **Тренер (Coach)** — интерактивный AI-ментор для подготовки и пост-анализа торговых сессий
- **Академия (Lesson)** — обучающие модули по методологии SMC/ICT
- **Сканер рынка** — инструмент поиска торговых возможностей на основе алгоритмического анализа

## ЧАСТЬ II: УСЛОВИЯ ИСПОЛЬЗОВАНИЯ AI-АССИСТЕНТА

### 9. Природа AI-рекомендаций

AI-Ассистент Jafar Pro использует большие языковые модели (LLM) для генерации аналитических отчётов. Вы признаёте и соглашаетесь, что:

- AI-модели могут допускать ошибки, галлюцинации и неточности
- Торговые уровни, прогнозы и рекомендации генерируются на основе вероятностных моделей и **не являются гарантированными**
- AI-Ассистент не обладает сознанием, интуицией или пониманием контекста, выходящего за рамки предоставленных данных
- Качество ответа AI зависит от качества и актуальности входных данных (рыночные котировки, данные стакана, новостной фон)

### 10. AI-Провайдеры

ПО может использовать следующие AI-модели:

| Провайдер | Модель | Флаг |
|---|---|---|
| DeepSeek | deepseek-chat / deepseek-v4 | `--deepseek` (по умолчанию) |
| Anthropic | Claude Opus 4.7 | `--claude` / `-c` |
| Google | Gemini 2.5 Pro | `--gemini` / `-g` |
| Ollama | Локальные модели | `--ollama` |

- При использовании облачных AI-провайдеров данные вашего запроса передаются на серверы соответствующего провайдера
- Пользователь соглашается с условиями использования соответствующего AI-провайдера
- При использовании Ollama (локально) данные не покидают устройство Пользователя
- Автор не контролирует и не несёт ответственности за обработку данных AI-провайдерами

### 11. Голосовой ассистент (Muxlisa)

- Голосовой вывод (`--voice`) активируется только по явному запросу Пользователя
- Озвучиваются только краткие сводки — не полный AI-анализ
- При использовании голосового вывода ПО использует локальный TTS-движок (macOS `say`)

## ЧАСТЬ III: КОНФИДЕНЦИАЛЬНОСТЬ И ДАННЫЕ

### 12. Сбор и обработка данных

**Данные, передаваемые внешним сервисам:**
- Рыночные запросы → TopstepX API, Yahoo Finance, Financial Modeling Prep
- AI-запросы → DeepSeek API, Anthropic API, Google Gemini API (в зависимости от выбранного провайдера)
- Telegram-уведомления → Telegram Bot API (номера сделок, баланс, PnL)

**Данные, хранимые локально:**
- API-ключи и токены → переменные окружения / `.env`
- Торговый журнал → `TRADING_JOURNAL.md`
- Конфигурация → конфигурационные файлы ПО

**Автор НЕ собирает, НЕ хранит и НЕ имеет доступа к:**
- Торговой истории Пользователя
- Данным торгового счёта
- API-ключам Пользователя
- Содержанию AI-запросов Пользователя

### 13. Telegram-уведомления

ПО может отправлять уведомления в Telegram (запуск, торговые сигналы, результаты сессий Coach). Уведомления содержат:
- Инструмент и торговое действие
- Баланс и PnL
- Торговые уровни
- Краткую аналитику

Telegram-уведомления **не являются** заменой интерфейса ПО и предоставляются «как есть». Задержка или отсутствие уведомления не является нарушением условий настоящего Соглашения.
